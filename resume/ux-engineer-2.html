5년차 개발자로 React와 TypeScript 진영에서 웹뷰 기반의 하이브리드 앱을 개발하고 있습니다.
현재는 째깍악어에서 FE팀을 리딩하며, 돌봄 예약/매칭 기반 커머스 플랫폼 째깍악어 하이브리드 앱을 개발·운영하고 있습니다.

빠른 실험과 데이터 기반 분석을 통해 디자인 시스템과 인터랙션을 일관된 구조로 설계하고, 서비스 전반의 UX 품질을 개선합니다.

깊게 분석하고 글쓰는 것을 즐겨테크 블로그를 5년 이상 운영하며 지식을 공유하고 있습니다.
또한 음향화 라이브러리@uturi/sonification 등 두 라이브러리의 메인테이너로 활동하고 있습니다.

더 자세한 프로젝트와 작업물은포트폴리오에서 확인하실 수 있습니다.

네이티브형 화면 전환 애니메이션 플러그인 개발 및 최적화

배경 및 전략

  • 하이브리드 앱 환경에서 모바일 사용자가 90% 이상을 차지하고 있어 웹뷰에서 네이티브에 준하는 화면 전환 경험이 요구되는 상황
  • 분절적인 화면 전환 경험 대신 사용자 여정에 따라서 다양한 방식으로 액티비티가 포개지는 화면 전환 효과 구현 결정

작업

  • @stackflow/core를 확장한 자체 전환 라이브러리 @dotss/plugin-screen-renderer 개발 및 배포
  • Next.js 환경 제약을 해결하기 위해 Streamable SSR 서버(Express 기반) 구축
  • 엣지/유니버셜 스와이핑 제스처를 구축, 시스템화하여 화면 전환 경험의 일관성과 예측 가능성 개선
  • 지연, 프리로드 및 리소스 서빙 최적화(lambda@edge) 전략 구축하여 서비스 최적화

결과

  • 전환 로직을 라이브러리 레벨로 추상화해 이후 화면 전환 UX를 일관되게 확장
  • 화면 전환 제스처에 대한 사용자 인터뷰 후 액티비티 역할별 제스처를 달리 적용하는 정책 수립
  • 관련 링크: 째깍악어(라이브) / 사용자 여정 분석을 통한 화면 전환 UX 개선 및 성능 최적화 / 유니버셜/엣지 스와이핑 분석과 서비스 도입

  • Vite 기반 번들링 환경에서 트리셰이킹 효율을 고려한 패키징 구성
  • 유닛 테스트(Vitest), 시각적 회귀 테스트(Chromatic), 접근성 테스트(axe-core)를 연계해 품질 검증 자동화
  • WCAG 2.2 AA 가이드라인을 기반으로 컴포넌트 단위 접근성 개선 및 가이드라인 생성
  • Streamable HTTP 기반 MCP 서버 구축하여 개발자 경험 및 기획~디자인~개발 프로세스 개선
